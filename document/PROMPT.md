# WebAI 会话启动 Prompt（精简版）

你是本仓库的新会话代码助手。先读本文件，再按下列规则执行。

## 1. 工作范围
- 重点目录：`JavaBackend/`、`python_backend/`
- 对照文档：`document/ARCHITECTURE.md`
- 忽略目录：`example-code/`

## 2. 执行原则
- 以源码与测试为准，文档仅用于对照。
- 先建立真实调用链，再下结论。
- 结论必须附文件路径。
- 不做无关的大重构。

## 3. 最小阅读流程
1. 读 `document/ARCHITECTURE.md`，提取目标分层。
2. 读 `JavaBackend/`：入口、服务、适配层、DTO。
3. 读 `python_backend/`：路由、服务、适配层、Schema。
4. 串联 `Java -> Python -> 模型提供方` 的输入/输出。
5. 用测试与配置核对行为、契约、错误处理是否一致。

## 4. 输出要求（不固定模板）
- 不强制固定章节或固定顺序。
- 根据问题复杂度组织输出，优先保证可执行性与可读性。
- 至少覆盖：当前结论、关键证据、主要问题（如有）、建议动作（如有）。
- 重要结论需附对应文件路径。

## 5. 冲突处理
- 文档与代码冲突：以代码为准，并标注冲突点。
- 有假设：显式写“假设”。
