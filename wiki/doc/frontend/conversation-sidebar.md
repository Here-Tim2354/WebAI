# ConversationSidebar 页面结构说明

这篇笔记帮助我们快速理解 `ConversationSidebar` 组件在页面中的职责，以及它在页面左侧栏里具体展示什么。

代码入口：
- `src/components/chat/conversation-sidebar.tsx`

关联笔记：
- [[conversation-sidebar-stateflow]]
- [[chatshell]]

## 本组件所用到的子组件

### Button

- 按钮基础组件
- 用于新建会话、展开收起、退出登录、确认删除等操作入口

### Badge

- 用于展示会话数量等小型状态标签

### Input

- 在会话重命名编辑态中作为标题输入框

### DropdownMenu

- 单条会话的操作菜单
- 用于重命名和删除入口

### Sheet

- 移动端抽屉式侧栏容器

### Dialog

- 删除会话确认弹窗

---

## 1. 组件职责

`ConversationSidebar` 是聊天工作区的左侧会话管理面板。

它负责的不是消息内容，而是会话导航和会话管理，包括：

- 展示历史会话列表
- 切换当前会话
- 新建会话
- 重命名会话
- 删除会话
- 收起 / 展开桌面端侧栏
- 打开 / 关闭移动端抽屉侧栏
- 退出登录

---

## 2. 顶层结构

组件最终返回三块内容：

```tsx
<>
  <aside>桌面端侧栏</aside>
  <Sheet>移动端侧栏</Sheet>
  <Dialog>删除确认弹窗</Dialog>
</>
```

可以把它理解成：

- `aside`：桌面端左侧栏
- `Sheet`：移动端左侧栏
- `Dialog`：删除动作的确认层

其中桌面端和移动端的主体内容不是各写一套，而是共享同一个 `sidebarContent`。

---

## 3. sidebarContent 共享内容块

组件内部先定义了一个：

```tsx
const sidebarContent = (...)
```

这块内容会被同时放进：

- 桌面端 `aside`
- 移动端 `SheetContent`

它的作用是：

- 保持两端会话区结构一致
- 避免维护两套相似 DOM

---

## 4. 顶部区域

`sidebarContent` 的第一部分是顶部信息区。

它包含：

- 工作区标识 `W`
- `Workspace / WebAI` 文案
- 当前用户邮箱
- 桌面端收起 / 展开按钮

作用是：

- 告诉用户当前所在工作区
- 显示当前登录用户
- 控制桌面端侧栏宽度

---

## 5. 新建会话入口

在顶部区域下方有“新对话”按钮。

这一块分成两种显示形态：

### 正常展开态

- 显示图标 + 文案“新对话”
- 桌面展开和移动端都可用

### 桌面收起态

- 只显示图标按钮
- 通过 `aria-label` 保留语义

这一块的主要职责是：

- 提供新建会话入口
- 在不同侧栏宽度下自适应展示方式

---

## 6. 会话列表区域

中间主体区域是会话列表。

它由两部分组成：

### 6.1 列表头

展示：

- 标题“最近对话”
- 当前会话数量 `Badge`

### 6.2 列表内容

如果没有会话：

- 显示空状态提示“还没有对话。”

如果有会话：

- 遍历 `conversations`
- 逐条渲染会话卡片

---

## 7. 单条会话项的结构

单条会话主要分两种状态：

### 7.1 编辑态

如果当前会话正在重命名：

- 渲染一个 `form`
- 内部放 `Input`
- 用户输入后提交标题

这是“局部替换成输入框”的编辑方式。

### 7.2 普通展示态

如果不是编辑态，会显示：

- 会话图标
- 会话标题
- 更新时间
- 操作菜单按钮

其中：

- 左边整个按钮区域用于切换会话
- 右边 `DropdownMenu` 用于打开操作菜单

---

## 8. 单条会话项的操作菜单

每条会话右侧都有菜单按钮。

菜单里目前包含：

- 重命名
- 删除

作用分别是：

- 重命名：进入编辑态，并把原标题放入 `titleDraft`
- 删除：打开删除确认弹窗

---

## 9. 底部状态区

侧栏最下方是状态区。

它包含：

- `Status` 文案
- 当前会话数量
- “当前工作区已连接 Supabase” 提示
- 退出登录按钮

它的作用是：

- 给用户一个工作区整体状态概览
- 提供退出登录入口

---

## 10. 桌面端与移动端的差异

### 桌面端

通过：

```tsx
<aside ...>
  {sidebarContent}
</aside>
```

展示固定侧栏。

特点：

- 可收起 / 展开
- 永远固定在左侧

### 移动端

通过：

```tsx
<Sheet open={mobileOpen} onOpenChange={onMobileOpenChange}>
  <SheetContent ...>
    {sidebarContent}
  </SheetContent>
</Sheet>
```

展示抽屉式侧栏。

特点：

- 由 `mobileOpen` 控制开关
- 从左侧滑出

---

## 11. 删除确认弹窗

组件返回的第三块是 `Dialog`。

它负责：

- 显示待删除会话的标题
- 告知删除后不可恢复
- 提供取消 / 确认删除按钮

这个弹窗不会直接展示在列表里，而是作为页面上的独立浮层出现。

---

## 12. 页面结构树

可以把 `ConversationSidebar` 压缩成下面的结构：

```tsx
ConversationSidebar
├─ sidebarContent
│  ├─ 顶部工作区信息区
│  ├─ 新建会话按钮
│  ├─ 会话列表头
│  ├─ 会话列表内容
│  │  └─ 会话项
│  │     ├─ 编辑态输入框
│  │     └─ 普通态按钮 + 菜单
│  └─ 底部状态区 + 退出登录
├─ aside（桌面端）
├─ Sheet（移动端）
└─ Dialog（删除确认）
```

---

## 13. 一句话理解

`ConversationSidebar` 本质上是工作区的会话导航与管理面板：它不负责聊天内容本身，而是负责“我现在在看哪个会话、有哪些会话、怎么管理这些会话”。
