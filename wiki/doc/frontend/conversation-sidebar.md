# ConversationSidebar 页面结构说明

这篇笔记帮助我们快速理解 `ConversationSidebar` 组件在页面中的职责，以及它在页面左侧栏里具体展示什么。

代码入口：
- `src/features/chat/components/conversation-sidebar.tsx`

关联笔记：
- [[conversation-sidebar-stateflow]]
- [[chatshell]]

## 本组件所用到的子组件

### Button

- 按钮基础组件
- 用于新建会话、展开收起、保存资料、上传头像、修改密码、退出登录和确认删除等操作入口

### Badge

- 用于展示会话数量等小型状态标签

### Input

- 在会话重命名编辑态中作为标题输入框

### DropdownMenu

- 单条会话的操作菜单
- 用于重命名和删除入口

### Sheet

- 移动端抽屉式侧栏容器

### Dialog

- 删除会话确认弹窗，以及个人账户、收藏区、归档区、Gemini 设置等二级面板

---

## 1. 组件职责

`ConversationSidebar` 是聊天工作区的左侧会话管理面板。

它负责的不是消息内容，而是会话导航和会话管理，包括：

- 展示历史会话列表
- 切换当前会话
- 新建会话
- 重命名会话
- 删除会话
- 收藏 / 取消收藏会话
- 打开收藏区
- 归档 / 恢复会话
- 打开归档区
- 管理个人账户资料
- 上传头像
- 修改登录密码
- 管理 Gemini Key、Base URL 和私有模型列表
- 收起 / 展开桌面端侧栏
- 打开 / 关闭移动端抽屉侧栏
- 退出登录

---

## 2. 顶层结构

组件最终返回三块内容：

```tsx
<>
  <aside>桌面端侧栏</aside>
  <Sheet>移动端侧栏</Sheet>
  <Dialog>删除确认与二级面板</Dialog>
</>
```

可以把它理解成：

- `aside`：桌面端左侧栏
- `Sheet`：移动端左侧栏
- `Dialog`：删除确认、个人账户、收藏区、归档区和 Gemini 设置等二级面板

其中桌面端和移动端的主体内容不是各写一套，而是共享同一个 `sidebarContent`。

---

## 3. sidebarContent 共享内容块

组件内部先定义了一个：

```tsx
const sidebarContent = (...)
```

这块内容会被同时放进：

- 桌面端 `aside`
- 移动端 `SheetContent`

它的作用是：

- 保持两端会话区结构一致
- 避免维护两套相似 DOM

---

## 4. 顶部区域

`sidebarContent` 的第一部分是顶部信息区。

它包含：

- WebAI 品牌图标
- `WebAI` 文案
- 桌面端收起 / 展开按钮

作用是：

- 告诉用户当前所在工作区
- 控制桌面端侧栏宽度

---

## 5. 新建会话入口

在顶部区域下方有“新对话”按钮。

这一块分成两种显示形态：

### 正常展开态

- 显示图标 + 文案“新对话”
- 桌面展开和移动端都可用

### 桌面收起态

- 只显示图标按钮
- 通过 `aria-label` 保留语义

这一块的主要职责是：

- 提供新建会话入口
- 在不同侧栏宽度下自适应展示方式

---

## 6. 会话列表区域

中间主体区域是会话列表。

它由两部分组成：

### 6.1 列表头

展示：

- 标题“最近对话”
- 当前会话数量 `Badge`

### 6.2 列表内容

如果没有会话：

- 显示空状态提示“还没有对话。”

如果有会话：

- 遍历 `conversations`
- 逐条渲染会话卡片
- 侧栏展开态使用固定宽度，列表项本身固定占满侧栏可用宽度
- 会话标题只在文本区域内截断，不反向撑开会话卡片或侧栏

---

## 7. 单条会话项的结构

单条会话主要分两种状态：

### 7.1 编辑态

如果当前会话正在重命名：

- 渲染一个 `form`
- 内部放 `Input`
- 用户输入后提交标题

这是“局部替换成输入框”的编辑方式。

### 7.2 普通展示态

如果不是编辑态，会显示：

- 会话图标
- 会话标题
- 更新时间
- 操作菜单按钮

其中：

- 左边整个按钮区域用于切换会话
- 标题区域使用 `min-w-0 flex-1` 承接剩余宽度
- 右边 `DropdownMenu` 用于打开操作菜单
- 长标题通过 `truncate` 显示，完整标题继续保留在按钮 `title` 中

---

## 8. 单条会话项的操作菜单

每条会话右侧都有菜单按钮。

菜单里目前包含：

- 重命名
- 收藏 / 取消收藏
- 归档
- 删除

作用分别是：

- 重命名：进入编辑态，并把原标题放入 `titleDraft`
- 收藏：把会话加入底部个人菜单中的收藏区
- 归档：把会话移出最近列表，后续可在归档区恢复
- 删除：打开删除确认弹窗

---

## 9. 底部用户区

侧栏最下方是用户入口。

它包含：

- 用户头像
- 用户名称
- 用户邮箱
- 个人账户入口
- 收藏区入口
- 归档区入口
- Gemini 设置入口
- 退出登录入口

它的作用是：

- 显示当前登录身份
- 承接个人资料、密码、头像和模型设置
- 承接收藏、归档等二级会话管理入口

---

## 10. 桌面端与移动端的差异

### 桌面端

通过：

```tsx
<aside ...>
  {sidebarContent}
</aside>
```

展示固定侧栏。

特点：

- 可收起 / 展开
- 永远固定在左侧

### 移动端

通过：

```tsx
<Sheet open={mobileOpen} onOpenChange={onMobileOpenChange}>
  <SheetContent ...>
    {sidebarContent}
  </SheetContent>
</Sheet>
```

展示抽屉式侧栏。

特点：

- 由 `mobileOpen` 控制开关
- 从左侧滑出

---

## 11. 二级面板

组件返回的第三块是 `Dialog` 体系。

它负责：

- 删除确认：显示待删除会话标题，并提供取消 / 确认删除按钮
- 个人账户：展示头像、用户名称、邮箱、昵称输入、头像上传和修改密码
- 收藏区：展示收藏会话并支持跳转
- 归档区：展示已归档会话并支持恢复
- Gemini 设置：保存 API Key / Base URL，拉取和管理私有模型列表

这些面板采用统一的宽面板、左对齐和低胶囊视觉，减少侧栏二级菜单之间的交互差异。

---

## 12. 页面结构树

可以把 `ConversationSidebar` 压缩成下面的结构：

```tsx
ConversationSidebar
├─ sidebarContent
│  ├─ 顶部工作区信息区
│  ├─ 新建会话按钮
│  ├─ 会话列表头
│  ├─ 会话列表内容
│  │  └─ 会话项
│  │     ├─ 编辑态输入框
│  │     └─ 普通态按钮 + 菜单
│  └─ 底部用户区 + 个人菜单
├─ aside（桌面端）
├─ Sheet（移动端）
└─ Dialog（删除确认 / 个人账户 / 收藏区 / 归档区 / Gemini 设置）
```

---

## 13. 一句话理解

`ConversationSidebar` 本质上是工作区的会话导航、会话组织和用户账户入口：它不负责聊天内容本身，而是负责“我现在在看哪个会话、有哪些会话、怎么管理会话和个人设置”。
