# AuthPanel 页面结构说明

这篇笔记帮助我们快速理解 `AuthPanel` 组件在页面中的职责，以及未登录时页面会展示什么。

代码入口：
- `src/features/chat/components/auth-panel.tsx`

关联笔记：
- [[auth-panel-stateflow]]
- [[chatshell]]

## 本组件所用到的子组件

### Input

- 邮箱和密码输入框

### Button

- 密码登录、发送登录链接和 GitHub 登录按钮

### Alert

- 用于展示发送成功、失败、过期等登录反馈

---

## 1. 组件职责

`AuthPanel` 是未登录用户进入系统时看到的登录面板。

它负责：

- 展示登录页视觉
- 接收邮箱输入
- 接收密码输入并发起邮箱密码登录
- 发送魔法链接登录请求作为备用入口
- 发起 GitHub OAuth 登录
- 展示登录反馈

---

## 2. 顶层结构

组件整体结构大致是：

```tsx
<motion.main>
  背景层
  <section>
    品牌标识
    标题与说明
    密码登录表单
    备用登录入口
    反馈提示
  </section>
</motion.main>
```

可以理解为：

- `main`：登录页容器
- 背景层：纯视觉氛围
- `section`：登录卡片主体

---

## 3. 品牌与标题区

卡片顶部会显示：

- `WebAI` 标识
- 标题“登录”
- 一句简短登录说明

这部分主要承担：

- 告知用户当前处于登录流程
- 统一品牌感

---

## 4. 表单区

登录表单由三部分组成：

- 邮箱输入框
- 密码输入框
- 登录按钮

邮箱输入框：

- 支持自动聚焦
- 显示邮件图标
- 使用受控输入方式

密码输入框：

- 显示锁图标
- 使用受控输入方式
- 与邮箱一起提交到 `/api/auth/password`

登录按钮：

- 默认显示“登录”
- 登录中显示进行中状态
- 成功后刷新到聊天工作区

备用入口：

- 发送 Magic Link
- GitHub 登录
- 两类备用入口与密码登录共用同一个反馈区

---

## 5. 反馈提示区

如果当前存在 `feedback`，页面下方会显示 `Alert`。

可能展示的内容包括：

- 登录链接已发送
- 密码登录失败
- GitHub 登录失败
- 登录确认失败
- 登录链接过期

这块是登录页的重要状态反馈区。

---

## 6. 一句话理解

`AuthPanel` 本质上是未登录态入口：默认走邮箱密码登录，同时保留 Magic Link 和 GitHub OAuth 作为备用方式。
