# 模型注册表扩展设想

需求来源：

- `phase_overview.md` 中 `Phase 4`
- 模型注册表、多模型能力扩展、统一 AI 兼容层

## 当前已成立的基础

当前已经验证成立的是：

- 使用 `ai_models` 作为父表
- 使用 `openai_compatible_models` 与 `gemini_models` 作为 provider 子表
- 上层统一映射为 `AIModel`
- 前端模型选择和服务端模型分发已经依赖这套注册表

## 当前仍属于扩展层的内容

以下内容应继续视为扩展字段或候选规划，不直接写进 `10_verified/`：

- 更完整的上游模型元数据镜像
- 调参模板和 provider 级默认参数
- 模型分组、展示分区和推荐策略
- 会话级默认模型快照
- 更多 provider 子表，如 `kimi`、`qwen` 等

## 当前建议

- 逻辑层继续把它视为一个统一模型注册表
- 物理层继续维持“父表 + provider 子表”
- 新 provider 优先通过新增子表接入，而不是继续在父表堆专属字段
